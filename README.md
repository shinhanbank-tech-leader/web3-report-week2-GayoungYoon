# 제목: 비트코인 결제와 기존 금융 지급결제의 구조적 차이 비교

## 부서명: 투자서비스개발부
## 이름: 윤가영

-------------------

### 1. 서론

기존 금융 시스템과 비트코인은 모두 가치 이전을 수행하지만, 거래를 기록하고 신뢰를 형성하는 방식에서 근본적으로 다르다. 기존 금융은 중앙기관이 관리하는 장부에 기반한 중앙집중 구조를 가지는 반면, 비트코인은 중앙기관 없이 분산 네트워크와 암호학적 검증을 통해 거래를 처리한다.

---

### 2. 기존 금융 지급결제 시스템

기존 금융 지급결제는 중앙기관이 관리하는 단일 원장을 기반으로 이루어진다. 국내에서는 한국은행이 최종 정산기관 역할을 수행하며, 거래 결과는 중앙 장부에 반영된 시점에 확정된다. 자산은 계좌 잔액 형태로 관리되는 계정 기반 모델을 사용하며, 거래는 잔액을 감소시키고 증가시키는 방식으로 처리된다.

동일 계좌에 대해 동시에 여러 거래가 발생할 경우 데이터 불일치가 발생할 수 있으므로 데이터베이스 잠금이나 원자적 처리 등을 통해 정합성을 유지한다. 정상적인 경우 승인된 거래는 즉시 확정되며 이후 번복될 가능성은 거의 없고, 오류 발생 시에는 금융기관의 개입을 통해 취소나 환불이 가능하다.

---

### 3. 비트코인 결제 시스템

비트코인은 중앙기관이 없는 분산 원장 시스템으로, 거래 기록은 네트워크 참여자들이 공동으로 유지한다. 자산은 계좌 잔액이 아니라 UTXO 모델에 따라 관리되며, 새로운 거래는 기존 UTXO를 소비하고 새로운 UTXO를 생성하는 방식으로 이루어진다. 이미 사용된 UTXO는 재사용할 수 없으므로 동일 자산의 중복 사용이 구조적으로 차단된다.

거래를 블록에 포함시키는 주체는 채굴자이며, 사용자는 거래 처리 우선순위를 높이기 위해 수수료를 지불한다. 수수료는 중앙기관이 아닌 채굴자에게 지급된다.

---

### 4. 거래 확정과 포크 가능성

중앙집중 원장에서는 거래가 승인되어 장부에 반영되는 순간 결제가 완료된 것으로 간주되며 거래 순서와 결과가 명확하게 결정된다. 반면 비트코인에서는 거래가 블록에 포함되더라도 즉시 최종 확정되지 않는다. 네트워크에서 동시에 블록이 생성되면 체인이 분기될 수 있으며, 이후 더 긴 체인이 선택되면서 일부 블록이 폐기될 수 있다. 이 경우 해당 블록에 포함된 거래는 다시 미확정 상태로 돌아갈 수 있으므로 먼저 전송된 거래가 반드시 먼저 확정된다고 보장할 수 없다. 추가 블록이 누적될수록 거래의 안정성이 높아지는 확률적 확정 구조를 가진다.

---

### 5. 결론

기존 금융 시스템은 중앙기관과 제도에 기반하여 높은 안정성과 즉시 확정성을 제공하는 반면, 비트코인은 분산 합의를 통해 중앙기관 없이 거래를 처리할 수 있지만 확정이 확률적으로 이루어진다. 두 시스템은 동일한 기능을 수행하면서도 신뢰의 근거와 운영 방식이 근본적으로 다르다.
